SMART COMMERCE AR - OMNICANALIDAD

Memoria Técnica:
Proyecto PIZHUN

Desarrollo de un prototipo funcional que une el Punto de Venta Físico con el Ecommerce. El cliente interactúa con un catálogo virtual, se prueba virtualmente la prenda mediante Face Tracking AR y accede directamente a la pasarela web.

01. Estrategia de Ecommerce (Tema 14)

PIZHUN es una marca ficticia de Streetwear Tecnológico. Nuestro producto principal es la PIZHUN Neural Cap (Gorra Táctica). El objetivo es que el cliente experimente el producto mediante un Virtual Try-On (probador virtual) y cierre la transacción en nuestro entorno web.

1.1

Estrategia de Precios (Pricing)

Para definir la viabilidad del producto en el mercado digital, se ha establecido la siguiente estructura de costes e impuestos basándonos en los requisitos del Tema 14:

25.00€
Coste Producción
40%
Margen de Beneficio
21%
IVA Aplicado
42.35€
Precio Final (PVP)

Justificación Estratégica: El coste de fabricación (materiales impermeables) es de 25€. Aplicando un margen de beneficio del 40% (10€ de ganancia neta), obtenemos una base imponible de 35€. Al sumar el 21% de IVA legal en España (7.35€), el precio final al consumidor es de 42.35€. Este precio es altamente competitivo en el nicho premium.

1.2

La Web como Capa de Confianza y Transacción

Mientras que Unity y la AR generan el efecto "wow", la landing page es la encargada de cerrar la venta. Se ha diseñado siguiendo una estricta filosofía Mobile-First:

  • Header & Hero: Logotipo visible, carrito y un CTA claro invitando a la compra tras la experiencia AR.
  • Ficha Técnica: Detalles "fríos" que no caben en la AR (materiales, impermeabilidad, tallaje).
  • Validación Legal: El Footer incluye políticas de devolución y métodos de pago seguros, elementos vitales para transmitir seguridad en un ecommerce.

02. Generación 2D (Imágenes y Vídeos) con Grok AI

Para cumplir con el criterio de "Calidad de Activos", utilizamos Grok Imagine de forma exclusiva para la generación visual 2D. Esto abarca las fotografías planas del producto base y los vídeos de fondo para la página web.

Prompt de Imagen (Fotografía Base 2D del Producto):

"A hyper-realistic frontal product shot of a futuristic tech-core streetwear cap, matte black waterproof fabric, tactical buckles, volumetric studio lighting, 8K resolution, clean product photography."
⚠ Alucinaciones en la IA de Vídeo
Obstáculo: Grok deformaba las prendas al generar el video de fondo web (formato .mp4) si le pedía "movimiento".
Solución: Modifiqué el prompt añadiendo "perfectly static subject" y "handheld camera movement". Al aislar la física y pedir que solo se moviese la cámara virtual, la imagen estática de la ropa mantuvo su coherencia estética en el vídeo.

03. Proceso Image-to-3D con Meshy

Meshy.ai fue el encargado de la conversión espacial. Tomó las imágenes planas generadas en Grok y aplicó el proceso Image-to-3D para generar los modelos volumétricos finales para Unity.

  1. Subimos la fotografía frontal (imagen 2D) generada en el paso anterior.
  2. Configuramos la exportación a High Quality (PBR Textures). Las texturas PBR son vitales para que la malla reaccione a la luz del mundo real.
  3. Exportamos el modelo tridimensional en formato .glb, el más optimizado para la memoria RAM de los móviles.
  4. Ajuste de Escala Crítico: Los modelos 3D generados por IA cargan con un tamaño colosal. En Unity seleccionamos este Prefab y le aplicamos una escala de 0.05 en los ejes X,Y,Z para que la prenda tenga un tamaño realista anatómico.

05. Desarrollo Técnico AR y UI (Tema 13: Unity)

Para este proyecto, se ha escalado la solución propuesta en la rúbrica. En lugar de un simple escaneo de imagen plana, se ha programado una arquitectura de interfaces (UI) completa y se ha utilizado Face Tracking para lograr un probador virtual realista.

Paso 1: Arquitectura de Menús (UI/Canvas Flow)

Para que la aplicación funcione como un catálogo real, se desarrolló el siguiente flujo de pantallas mediante UI Canvas:

  1. Menú de Productos: Al abrir la App, el usuario ve un Canvas con la lista de productos disponibles.
  2. Pop-up de Detalle: Al seleccionar un producto, se activa un panel con la información del artículo y un botón de "Probar Virtualmente".
  3. Escena AR (Overlay): Al pulsar el botón, se activa la cámara frontal AR. Encima del feed del mundo real, coloqué un Canvas Overlay que contiene dos botones estáticos: "Volver al Menú" y "Comprar en Web".

Paso 2 y 3: Configuración AR Foundation (Face Tracking)

Evolución de la rúbrica: Dado que el producto es una gorra, el uso de "Image Tracking" resulta poco inmersivo. Por ello, se implementó el sistema AR Face Manager para anclar la malla a la anatomía del usuario.

  1. Instalación: En Window > Package Manager, instalamos AR Foundation y el ARCore XR Plugin.
  2. Setup de Escena (Face Tracking): Eliminamos la cámara tradicional e introdujimos el AR Session Origin. En lugar del Image Tracker, añadimos el script AR Face Manager.
  3. A este Manager se le asigna nuestro Prefab de la gorra. El motor de Unity detectará la malla de la cara del usuario a través de la cámara frontal y superpondrá la prenda.

Paso 4: Programación de Controladores (C#)

Para lograr la interacción fluida entre la cámara, los modelos y la web, se desarrollaron scripts avanzados depurados con IA, comentados exhaustivamente.

Prompt de Programación (ChatGPT):

"En Unity con AR Foundation, estoy usando Face Tracking para poner una gorra. 1) Necesito un script (ARHatController.cs) que mueva el modelo hacia arriba porque el punto ancla por defecto está en la nariz. 2) Necesito un script para gestionar mi Canvas UI, incluyendo un método para abrir mi URL y otro para recargar la escena de menú, mejorando la UX en lugar de obligar al usuario a tocar el modelo 3D. Comenta cada línea."

1. ARHatController.cs (Solución al Offset Anatómico)

// Librerías necesarias para la Realidad Aumentada Facial using UnityEngine; using UnityEngine.XR.ARFoundation; // Obliga a que el objeto tenga un ARFace adosado [RequireComponent(typeof(ARFace))] public class ARHatController : MonoBehaviour { private ARFace _arFace; private GameObject _currentHatInstance; void Awake() { _arFace = GetComponent<ARFace>(); } void OnEnable() { _arFace.updated += OnFaceUpdated; } void OnDisable() { _arFace.updated -= OnFaceUpdated; } // Se ejecuta constantemente evaluando si la cámara capta el rostro private void OnFaceUpdated(ARFaceUpdatedEventArgs args) { if (_currentHatInstance != null) { if (_arFace.trackingState == UnityEngine.XR.ARSubsystems.TrackingState.Tracking) _currentHatInstance.SetActive(true); else _currentHatInstance.SetActive(false); } } public void SetHatPrefab(GameObject newHatPrefab) { if (_currentHatInstance != null) Destroy(_currentHatInstance); if (newHatPrefab != null) { _currentHatInstance = Instantiate(newHatPrefab); _currentHatInstance.transform.SetParent(transform, false); _currentHatInstance.transform.localScale = Vector3.one; // EL GRAN ARREGLO ESPACIAL: // El motor sitúa el ancla en el centro del cráneo (nariz). // Este offset sube la gorra 15cm (Y) y la adelanta 5cm (Z) hacia la frente. _currentHatInstance.transform.localPosition = new Vector3(0, 0.15f, 0.05f); _currentHatInstance.transform.localRotation = Quaternion.identity; } } }

2. HologramGlitchAnimator.cs (Estética UI)

// Gestiona los efectos de parpadeo simulando un sistema antiguo al abrir la interfaz. using System.Collections; using UnityEngine; using UnityEngine.UI; [RequireComponent(typeof(CanvasGroup))] public class HologramGlitchAnimator : MonoBehaviour { private CanvasGroup _canvasGroup; private RectTransform _rectTransform; public float transitionDuration = 0.4f; public float startScaleY = 0.05f; private Vector3 _originalScale; private void Awake() { _canvasGroup = GetComponent<CanvasGroup>(); _rectTransform = GetComponent<RectTransform>(); _originalScale = Vector3.one; } private void OnEnable() { // Inicia la rutina de animación de glitch paralela StartCoroutine(GlitchInCoroutine()); } private IEnumerator GlitchInCoroutine() { float elapsed = 0f; _canvasGroup.alpha = 0.2f; // Bucle que altera la opacidad y escala durante 0.4 segundos while (elapsed < transitionDuration) { elapsed += Time.unscaledDeltaTime; float t = elapsed / transitionDuration; if (Random.value > t) _canvasGroup.alpha = Random.Range(0.1f, 0.8f); else _canvasGroup.alpha = Mathf.Lerp(0.5f, 1f, t); yield return null; } // Estabiliza el panel al final de la animación _canvasGroup.alpha = 1f; _rectTransform.localScale = _originalScale; } }
UX/UI Mejora: Sustitución de OnMouseDown por Botones Canvas
Decisión de Diseño: La rúbrica inicial sugería usar un Box Collider en la malla y el evento OnMouseDown(). Sin embargo, dado que el usuario se está probando la gorra moviendo la cabeza, intentar "tocar" un modelo 3D en movimiento resultaba frustrante.
Solución: Se implementaron botones clásicos de Canvas UI Overlay (Volver al Menú e Ir a Web) con métodos públicos, garantizando un salto limpio y accesible hacia nuestro Ecommerce web.

Paso 5: Compilación y Exportación (Android Build)

  1. Ajustes de Rendering: En Project Settings > Player, forzamos el uso de OpenGLES3 y quitamos Vulkan (que rompe el feed de cámara en Android).
  2. Build Settings: En File > Build Settings seleccionamos Android y pulsamos Switch Platform.
  3. Incluimos ambas escenas (MenuPrincipal y EscenaAR) usando Add Open Scenes.
  4. Pulsamos Build, obteniendo el .apk final para la demostración en vídeo.
