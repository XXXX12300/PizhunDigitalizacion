SMART COMMERCE AR - OMNICANALIDAD

Memoria Técnica:
Proyecto PIZHUN

Desarrollo de un prototipo funcional que une el Punto de Venta Físico con el Ecommerce. El cliente escanea la etiqueta de una prenda, visualiza el modelo inmersivo en Realidad Aumentada y accede directamente a la pasarela de pago web.

01. Estrategia de Ecommerce (Tema 14)

PIZHUN es una marca ficticia de Streetwear Tecnológico. Nuestro producto principal es la PIZHUN Neural Cap (Gorra Táctica). El objetivo de la Omnicanalidad es que el cliente en la tienda física experimente el producto mediante AR y cierre la transacción en nuestro entorno digital.

1.1

Estrategia de Precios (Pricing)

Para definir la viabilidad del producto en el mercado digital, se ha establecido la siguiente estructura de costes e impuestos basándonos en los requisitos del Tema 14:

25.00€
Coste Producción
40%
Margen de Beneficio
21%
IVA Aplicado
42.35€
Precio Final (PVP)

Justificación Estratégica: El coste de fabricación (materiales impermeables) es de 25€. Aplicando un margen de beneficio del 40% (10€ de ganancia neta), obtenemos una base imponible de 35€. Al sumar el 21% de IVA legal en España (7.35€), el precio final al consumidor es de 42.35€. A nivel de marketing digital, este precio es altamente competitivo en el nicho del "Tech-Wear" premium, permitiendo absorber los costes de la infraestructura de Realidad Aumentada.

1.2

La Web como Capa de Confianza y Transacción

Mientras que Unity y la AR generan el efecto "wow", la landing page es la encargada de cerrar la venta. Se ha diseñado siguiendo una estricta filosofía Mobile-First:

  • Header & Hero: Logotipo visible, carrito y un CTA claro invitando a la compra tras la experiencia AR.
  • Ficha Técnica: Detalles "fríos" que no caben en la AR (materiales, impermeabilidad, tallaje).
  • Validación Legal: El Footer incluye políticas de devolución y métodos de pago seguros, elementos vitales para transmitir seguridad en un ecommerce.

02. Calidad de Activos: Ideación con Grok AI

Para cumplir con el criterio de "Calidad de Activos" de la rúbrica, usamos Grok Imagine para generar el arte conceptual del producto y los fondos de la web.

Prompt de Imagen (Generación del Producto):

"A hyper-realistic frontal product shot of a futuristic tech-core streetwear cap, matte black waterproof fabric, tactical buckles, volumetric studio lighting, 8K resolution, clean product photography."
⚠ Alucinaciones en la IA de Vídeo
Obstáculo: Grok deformaba las prendas al generar el video de fondo web si le pedía "movimiento".
Solución: Modifiqué el prompt añadiendo "perfectly static subject" y "handheld camera movement". Al aislar la física, la ropa mantuvo su coherencia estética.

03. Modelado 3D GenAI con Meshy

Meshy.ai convirtió las fotografías planas en mallas volumétricas 3D para la app de Unity.

  1. Subimos la fotografía frontal recortada.
  2. Configuramos la exportación a High Quality (PBR Textures). Las texturas PBR (Physically Based Rendering) son vitales en AR Foundation para que la gorra reaccione a la luz del mundo real.
  3. Exportamos en formato .glb, el más optimizado para la memoria RAM de los smartphones. Aunque convertimos a OBJ
  4. Ajuste de Escala Crítico: Los modelos de IA salen gigantes. Como exige la rúbrica, en Unity seleccionamos el Prefab y aplicamos una escala de 0.05 en X,Y,Z para que la gorra tenga un tamaño real y no colapse la cámara.

05. Desarrollo Técnico AR (Tema 13: Unity)

Esta es la guía ultra-detallada de cómo se construyó la experiencia omnicanal usando Unity 2022.3 LTS y el framework gratuito AR Foundation.

Paso 1 y 2: Instalación y Package Manager

  1. Descarga del Motor: A través de Unity Hub, instalamos la versión Unity 2022.3 LTS e iniciamos un nuevo proyecto bajo la plantilla 3D (Core).
  2. Instalación de Librerías: Navegamos a Window > Package Manager. Cambiamos el registro a Unity Registry y buscamos e instalamos el paquete base AR Foundation. Acto seguido, instalamos el ARCore XR Plugin (obligatorio para compilar la app en dispositivos Android).
  3. Configuración XR: Fuimos a Edit > Project Settings > XR Plug-in Management. En la pestaña con el icono del androide, marcamos la casilla "ARCore" para habilitar los sensores de la cámara del teléfono en la compilación.
  4. Limpieza de Escena: En la ventana Hierarchy, borramos la Main Camera tradicional. Haciendo clic derecho y navegando al submenú XR, añadimos los componentes vitales: AR Session (gestiona el ciclo de vida de la AR) y AR Session Origin (transforma las coordenadas reales al espacio de Unity).

Paso 3: El Sistema de Rastreo (Marcador a Modelo)

Nota Técnica: Para cumplir con la rúbrica, el flujo se inicia escaneando la etiqueta física de la prenda (Image Tracking), la cual desencadena la aparición del modelo virtual interactivo.

  1. Configuración del Marcador: Seleccionamos el AR Session Origin y en el Inspector añadimos el script AR Tracked Image Manager.
  2. Reference Image Library: Creamos una base de datos de imágenes en nuestros Assets. Subimos la foto de la etiqueta de PIZHUN. Nos aseguramos de que el diseño tuviera un alto contraste y formas asimétricas, logrando que Unity nos otorgara una calificación de rastreo óptima (superior a 3 estrellas) para evitar que el modelo "baile".
  3. Asignación del Prefab: Arrastramos nuestro modelo .glb de la gorra (previamente escalado a 0.05) a la casilla "Tracked Image Prefab". Al detectar la etiqueta física en la tienda, Unity superpondrá este modelo automáticamente.
  4. Colisionadores Físicos: Requisito Indispensable: Para que la pantalla táctil detecte nuestro dedo, seleccionamos el Prefab de la gorra y le añadimos un Box Collider en el Inspector. Sin esto, el Raycast de la pantalla lo ignoraría por completo.

Paso 4: Programación de la Interacción (C#)

Cumpliendo con la rúbrica, se detalla el script que une Unity con el Ecommerce, incluyendo comentarios exhaustivos en cada línea generados y depurados con ayuda de ChatGPT.

Prompt de Programación (ChatGPT):

"En Unity 2022.3 con AR Foundation, tengo un Prefab 3D con un Box Collider que aparece al leer un marcador. Necesito un script en C# llamado BotonCompra.cs que, al tocar el modelo en la pantalla táctil, abra una URL (mi ecommerce). Comenta cada línea del código detalladamente explicando su función para una memoria universitaria."
// Importamos la librería principal del motor Unity que nos permite interactuar con el ecosistema y componentes. using UnityEngine; // Definimos la clase pública BotonCompra que hereda de MonoBehaviour (necesario para adosarlo a un GameObject). public class BotonCompra : MonoBehaviour { // Creamos una variable pública de tipo texto (string). Al ser pública, podemos pegar el link de nuestro // Ecommerce cómodamente desde la ventana del Inspector de Unity sin tener que volver a abrir el código. public string urlCompra = "https://proyecto-pizhun.com/checkout"; // OnMouseDown es un método nativo de las físicas de Unity. // Se ejecuta automáticamente cuando el usuario toca (o hace clic) sobre la pantalla del móvil, // lanzando un Raycast invisible que choca contra el "Box Collider" que le pusimos a nuestro modelo 3D. void OnMouseDown() { // Application.OpenURL invoca la API del Sistema Operativo del teléfono (Android o iOS). // Suspende la cámara de AR y abre el navegador web por defecto directamente en nuestra pasarela de pago. Application.OpenURL(urlCompra); // Imprime un mensaje en la consola de desarrollo de Unity para que sepamos que el toque ha funcionado // mientras estamos haciendo pruebas en el ordenador antes de compilar. Debug.Log("Redirigiendo a la pasarela del Ecommerce..."); } }

Paso 5: Compilación y Exportación (Android Build)

  1. Ajustes de Rendering: En Project Settings > Player, deshabilitamos la API gráfica "Vulkan" y forzamos el uso de OpenGLES3. Vulkan causa un error conocido de "pantalla azul" con el feed de la cámara de ARCore.
  2. Build Settings: Navegamos a File > Build Settings. Seleccionamos la plataforma Android y pulsamos Switch Platform (Unity re-importa todos los assets adaptándolos a móvil).
  3. Pulsamos Add Open Scenes para asegurar que nuestra escena de AR esté incluida.
  4. Finalmente, pulsamos Build. Elegimos una carpeta en nuestro PC y Unity nos generó el archivo .apk listo para instalar y grabar el vídeo de demostración solicitado en la entrega.
