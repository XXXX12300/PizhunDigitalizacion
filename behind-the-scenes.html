SMART COMMERCE AR - OMNICANALIDAD

Memoria Técnica:
Proyecto PIZHUN

Desarrollo de un prototipo funcional que digitaliza la experiencia de compra. El cliente abre la aplicación, interactúa con un catálogo virtual, se prueba virtualmente la prenda mediante Face Tracking AR y accede directamente a la pasarela web de pago con un solo toque.

01. Estrategia de Ecommerce (Tema 14)

PIZHUN es una marca ficticia de Streetwear Tecnológico. Nuestro producto principal es la PIZHUN Neural Cap (Gorra Táctica). El objetivo es que el cliente experimente el producto mediante un Virtual Try-On (probador virtual inmersivo) desde su propio dispositivo y cierre la transacción en nuestro entorno web seguro.

Justificación Estratégica: El coste de fabricación (materiales impermeables) es de 25€. Aplicando un margen de beneficio del 40% (10€ de ganancia neta), obtenemos una base imponible de 35€. Al sumar el 21% de IVA legal en España (7.35€), el precio final al consumidor es de 42.35€. Este precio es altamente competitivo en el nicho premium, absorbiendo los costes de la plataforma AR.

1.2

La Web como Capa de Confianza y Transacción

Mientras que la aplicación de Unity genera el efecto "wow" del probador, la landing page web es la encargada de cerrar la venta. Se ha diseñado siguiendo una estricta filosofía Mobile-First:

  • Header & Hero: Logotipo visible, carrito y un CTA claro invitando a la compra tras la experiencia AR.
  • Ficha Técnica: Detalles "fríos" que no caben en la AR (composición de materiales, impermeabilidad, tallaje).
  • Validación Legal: El Footer incluye políticas de devolución y métodos de pago seguros, elementos vitales para transmitir seguridad en un ecommerce moderno.

02. Generación 2D (Imágenes y Vídeos) con Grok AI

Para cumplir con el criterio de "Calidad de Activos", utilizamos Grok Imagine de forma exclusiva para la generación visual 2D. Esto abarca las fotografías planas del producto base y los vídeos de fondo para la página web.

Prompt de Imagen (Fotografía Base 2D del Producto):

"A hyper-realistic frontal product shot of a futuristic tech-core streetwear cap, matte black waterproof fabric, tactical buckles, volumetric studio lighting, 8K resolution, clean product photography."
⚠ Alucinaciones en la IA de Vídeo
Obstáculo: Grok deformaba las prendas al generar el video de fondo web (formato .mp4) si le pedía "movimiento".
Solución: Modifiqué el prompt añadiendo "perfectly static subject" y "handheld camera movement". Al aislar la física y pedir que solo se moviese la cámara virtual, la imagen estática de la ropa mantuvo su coherencia estética en el vídeo generado.

03. Proceso Image-to-3D con Meshy

Meshy.ai fue el encargado de la conversión espacial. Tomó las imágenes planas generadas en Grok y aplicó el proceso Image-to-3D para generar los modelos volumétricos finales para Unity.

  1. Subimos la fotografía frontal (imagen 2D) generada en el paso anterior.
  2. Configuramos la exportación a High Quality (PBR Textures). Las texturas PBR son vitales para que la malla reaccione a la luz del mundo real en la experiencia de Realidad Aumentada.
  3. Exportamos el modelo tridimensional en formato .glb, el más optimizado para la memoria RAM de los dispositivos móviles.
  4. Ajuste de Escala Crítico: En Unity seleccionamos este Prefab y le aplicamos una escala matemática de 0.05 en los ejes X,Y,Z para que la prenda tenga un tamaño realista anatómico y encaje con la cabeza del usuario.

05. Desarrollo Técnico AR y UI (Tema 13: Unity)

En este proyecto, se ha evolucionado la rúbrica base para crear un producto comercial realista. En lugar de forzar al usuario a imprimir un marcador físico estático, se ha implementado Face Tracking AR (Probador Virtual) y una arquitectura de interfaces gráficas completas (Canvas UI).

Paso 1: Jerarquía y Flujo UI (Canvas Flow)

Para que la aplicación funcione como un catálogo real, se estructuró la jerarquía de Unity separando la interfaz de usuario de la sesión de AR:

Captura real de la jerarquía (Hierarchy) del proyecto en Unity.

  1. Menú de Productos: Al abrir la App, el usuario interactúa con el Canvas que contiene el catálogo y los detalles del producto (Pop-up).
  2. AR Session Origin: Contiene la cámara AR y el AR Face Manager que se activa al pulsar "Probar".
  3. Canvas Overlay (AR_Canvas): Superpuesto al feed de la cámara, contiene dos botones estáticos: "Volver" y "Go Web".
Mejora UX/UI frente a la Rúbrica Base
Evolución del Tracking: Dado que vendemos una gorra, usar "Image Tracking" en una mesa no aporta valor. Implementamos AR Face Manager para anclar la prenda a la anatomía del usuario (Try-On).

Evolución de Interfaz: La rúbrica sugería colocar un BoxCollider en el modelo 3D. Dado que la gorra se mueve dinámicamente con la cabeza, obligar al usuario a atinarle con el dedo provoca mala experiencia. Por ello, programamos botones de Canvas UI estáticos para garantizar un salto cómodo hacia la web.

Paso 2: Programación de Controladores (C#)

Scripts avanzados depurados con IA, comentados de forma exhaustiva para esta memoria.

1. ARHatController.cs (Solución al Offset Anatómico)

// Librerías necesarias para usar las físicas y la Realidad Aumentada de Unity using UnityEngine; using UnityEngine.XR.ARFoundation; // Exige al motor que el objeto que lleve este script tenga un componente ARFace adosado [RequireComponent(typeof(ARFace))] public class ARHatController : MonoBehaviour { private ARFace _arFace; private GameObject _currentHatInstance; void Awake() { _arFace = GetComponent<ARFace>(); } void OnEnable() { _arFace.updated += OnFaceUpdated; } void OnDisable() { _arFace.updated -= OnFaceUpdated; } // Función que se ejecuta en bucle evaluando la posición del rostro private void OnFaceUpdated(ARFaceUpdatedEventArgs args) { if (_currentHatInstance != null) { if (_arFace.trackingState == UnityEngine.XR.ARSubsystems.TrackingState.Tracking) _currentHatInstance.SetActive(true); else _currentHatInstance.SetActive(false); } } public void SetHatPrefab(GameObject newHatPrefab) { if (_currentHatInstance != null) Destroy(_currentHatInstance); if (newHatPrefab != null) { _currentHatInstance = Instantiate(newHatPrefab); _currentHatInstance.transform.SetParent(transform, false); _currentHatInstance.transform.localScale = Vector3.one; // EL GRAN ARREGLO ESPACIAL DE UX: // Unity sitúa el ancla en el centro de la cabeza. // Este 'offset' desplaza la gorra 15cm arriba (Y) y 5cm adelante (Z). _currentHatInstance.transform.localPosition = new Vector3(0, 0.15f, 0.05f); _currentHatInstance.transform.localRotation = Quaternion.identity; } } }

3. HologramGlitchAnimator.cs (Estética UI en Menús)

// Gestiona los efectos de parpadeo simulando un sistema antiguo al abrir la app. using System.Collections; using UnityEngine; using UnityEngine.UI; [RequireComponent(typeof(CanvasGroup))] public class HologramGlitchAnimator : MonoBehaviour { private CanvasGroup _canvasGroup; private RectTransform _rectTransform; public float transitionDuration = 0.4f; private Vector3 _originalScale; private void Awake() { _canvasGroup = GetComponent<CanvasGroup>(); _rectTransform = GetComponent<RectTransform>(); _originalScale = Vector3.one; } private void OnEnable() { StartCoroutine(GlitchInCoroutine()); } private IEnumerator GlitchInCoroutine() { float elapsed = 0f; _canvasGroup.alpha = 0.2f; while (elapsed < transitionDuration) { elapsed += Time.unscaledDeltaTime; float t = elapsed / transitionDuration; if (Random.value > t) _canvasGroup.alpha = Random.Range(0.1f, 0.8f); else _canvasGroup.alpha = Mathf.Lerp(0.5f, 1f, t); yield return null; } _canvasGroup.alpha = 1f; _rectTransform.localScale = _originalScale; } }
