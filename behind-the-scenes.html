01. Estrategia de Diseño y Flujo de Usuario (Tema 14)

El objetivo de este trabajo es demostrar cómo una experiencia física en el mundo real puede conectarse fluidamente con un entorno digital web interactivo. El flujo deseado es: el usuario interactúa con la prenda física mediante la cámara de su móvil (Unity AR) y, al tocarla, un Deep Link lo lanza a la web para visualizar el producto en detalle.

02. Ideación Visual y Audiovisual con Grok

Grok Imagine fue el motor elegido para generar texturas hiperrealistas y los fondos de vídeo atmosféricos.

  1. Prompt Base de Imagen (Para prendas):
    "A hyper-realistic frontal product shot of a futuristic tech-core streetwear cap, matte black waterproof fabric, tactical buckles, volumetric studio lighting, 8K resolution, clean product photography."
  2. Prompt de Vídeo Cinemático (Fondos web):
    "Slow cinematic pan, perfectly static subject, handheld camera movement over a cyberpunk streetwear jacket, heavy rain environment, shallow depth of field, neon reflections."
⚠ Alucinaciones en la IA
Obstáculo: Grok deformaba las prendas al generar el video si le pedía "movimiento".
Solución: Modifiqué las directrices añadiendo "perfectly static subject". Al indicar que el objeto debía estar rígido y solo la cámara debía moverse, aislé el movimiento y evité las alucinaciones de texturas.

03. Modelado 3D GenAI con Meshy

Meshy.ai convirtió las fotografías planas de Grok en mallas volumétricas `.glb` para inyectarlas en la aplicación de Unity.

  1. Subimos la fotografía plana frontal (limpia sin fondo).
  2. Configuramos la exportación obligatoria a High Quality (PBR Textures). Es vital para generar mapas de rugosidad y normales (Normal Maps) para que Unity pueda simular cómo la luz rebota en la tela de forma realista.
  3. Exportamos en formato .glb (GL Transmission Format), que empaqueta texturas y mallas en un solo archivo binario muy optimizado para RAM de móviles.
⚠ Escala Absoluta World-Space
Obstáculo: El `.glb` cargaba midiendo 10 metros en Unity, colapsando el rastreo de cámara.
Solución: Meshy computa sin métrica, Unity usa 1 unidad = 1 metro. Modifiqué manualmente el Scale en Unity a 0.05 en los ejes X,Y,Z reduciendo el modelo a proporciones anatómicas de una gorra real.

04. Arquitectura Web y Antigravity IDE

La programación de la interfaz (Front-End) se ejecutó íntegramente en el ecosistema en la nube Antigravity IDE, gestionando estructuras HTML complejas, interactividad y estilos.

Ingeniería del DOM y Layout de Producto

  1. Incrustación de Modelos de Grok: Para lograr la atmósfera inmersiva, tomé el `.mp4` generado por Grok y lo declaré como fondo en la capa más profunda del documento:
    <video class="bg-layer" autoplay loop muted playsinline src="vid/grok_bg.mp4">.
    Mediante CSS le asigné position: fixed; z-index: -1; y un filtro de oscurecimiento filter: brightness(0.3); para garantizar que el texto blanco fuera legible.
  2. Secciones y Rejillas de Producto: Para exhibir las prendas sin usar librerías externas de carruseles, programé el contenedor usando CSS Grid Nativo. La regla grid-template-columns: repeat(auto-fit, minmax(300px, 1fr)) garantizó que, al verse en un móvil en vertical, las tarjetas de ropa formaran una sola columna, pero en un iPad o PC se auto-expandieran a 2 o 3 columnas.
  3. Sistema de Pop-ups Interactivos: Para mostrar detalles técnicos de la ropa al hacer clic en "Ver más", evité manipular opacidades en divs vacíos y utilicé la moderna API `` de HTML5.
    Creé una etiqueta <dialog id="product-modal">. Esto permite que el pop-up se posicione nativamente por encima del DOM (Top-Layer) bloqueando el scroll del fondo automáticamente. Se activa con una simple orden Javascript: document.getElementById('product-modal').showModal();.
⚠ Cuellos de Botella (Lag y Stuttering en GSAP)
Al enlazar los eventos de ScrollTrigger y el seguimiento del ratón sobre los vídeos pesados de Grok, el navegador recalentaba y perdía Frames.
Descubrí que animar márgenes forzaba un "Reflow". Se refactorizó usando la API gsap.quickTo(). Esta función esquiva el DOM de CSS y envía las coordenadas de interpolación X/Y directamente a la GPU, logrando transiciones mantecosas.

05. Desarrollo de Realidad Aumentada (Unity 3D)

La conexión física a digital se programó en Unity, gestionando la cámara a través del SDK de ARCore de Google.

Configuración, Package Manager y Deep Linking

  1. Instalación de Paquetes: En Window > Package Manager, en el Unity Registry, instalamos los motores troncales: AR Foundation y ARCore XR Plugin (para Android).
  2. Ajustes de Rendering: En Player Settings, eliminamos la API gráfica "Vulkan" y forzamos OpenGLES3 para evitar corrupciones de buffer de cámara.
  3. Setup de Escena (Image Tracking): Eliminamos la cámara normal e introdujimos el AR Session Origin. Añadimos el script AR Tracked Image Manager, cargándole un marcador de alto contraste para lograr un Tracking Score excelente.
  4. Instalación Física Interactiva: Al modelo volumétrico exportado de Meshy le adosamos un componente Box Collider. Sin este colisionador, el Raycast de la pantalla táctil atravesaría la malla como si fuera un fantasma, haciendo imposible el toque.
⚠ Generación de la Compilación (Android Build .apk)
Para transformar el proyecto en una App real de móvil, navegamos a File > Build Settings, seleccionamos la arquitectura Android y ejecutamos un Switch Platform para recompilar los recursos. Al pulsar Build, obtuvimos nuestro archivo .apk ejecutable de forma nativa.

BotonCompra.cs - Enlace C# con la Web

El script vital insertado en el Box Collider del modelo AR. Al detectar el tap en la pantalla del smartphone, suspende Unity y dispara un Deep Link hacia nuestro dominio web para mostrar el detalle del producto.

using UnityEngine; public class BotonCompra : MonoBehaviour { // URL del proyecto enlazada directamente public string urlProyectoWeb = "https://nuestro-proyecto-pizhun.com/entrada"; // Evento nativo de colisión (Touch en AR) void OnMouseDown() { // Interfaz de sistema que abre el navegador nativo del móvil Application.OpenURL(urlProyectoWeb); Debug.Log("Saliendo de la simulación AR, abriendo plataforma web..."); } }
