Un recorrido paso a paso, con extremo detalle, sobre el proceso de diseño y desarrollo de PIZHUN. Construido como proyecto académico explorando la intersección del diseño web y la Inteligencia Artificial interactiva.

01. Estrategia de Diseño y Flujo de Usuario (Tema 14)

El objetivo de este proyecto de clase es explorar cómo una experiencia física puede conectarse fluidamente con un entorno digital interactivo. PIZHUN actúa como un concepto académico de marca de Streetwear Tecnológico (Tech-Core). Queremos que el usuario interactúe con el elemento físico mediante la cámara de su móvil y salte directamente a una landing page web inmersiva.

1.1

Inspiración y Creación del Concepto (Prompting Web)

Para establecer las bases de la interfaz web, utilizamos como gran inspiración la interactividad y estética mostrada en este vídeo de referencia conceptual. El reto era traducir esa fluidez a código Front-End real.

Para no contaminar la memoria contextual de la IA con otros proyectos, utilicé ChatGPT explícitamente en modo incógnito para generar la estructura inicial y refinar los estilos de la web de forma aislada. Esto garantizó respuestas mucho más precisas y enfocadas únicamente en el diseño "Tech-Wear".

Prompt Web en ChatGPT (Incógnito):
"Actúa como un desarrollador Front-End experto y UI/UX designer. Necesito que estructures el esqueleto HTML5 y el CSS3 base para una landing page de estilo 'Tech-Wear' cyberpunk minimalista. Debe ser completamente Mobile-First, usar variables CSS para un tema oscuro (negros mate y acentos neón morados), y estar preparada para integrar animaciones complejas con la librería GSAP. Omite Javascript por ahora, solo dame una maqueta semántica perfecta."
1.2

La Web como Capa de Interacción

La web desarrollada sirve como el núcleo informativo y de conversión del proyecto. Mientras que Unity genera el efecto visual en el mundo real, la web recoge al usuario.

  1. Estructura Mobile-First Rígida: Como el usuario viene inevitablemente de interactuar con la Realidad Aumentada desde su teléfono móvil, se han usado Media Queries, CSS Grid avanzado y Flexbox para garantizar una navegación ergonómica a una sola mano.
  2. Semántica HTML5 y Accesibilidad: Uso riguroso de etiquetas como <header>, <main>, y <article> para asegurar que el DOM es legible, manteniendo un contraste de colores validado por las normativas W3C.
  3. Tensión Visual: Implementación de estados :hover y micro-interacciones sutiles para guiar el recorrido del ojo del usuario a lo largo del trabajo.

02. Ideación Visual y Audiovisual con Grok

Grok Imagine fue el núcleo creativo para los assets. El objetivo era deconstruir la estética tradicional del "streetwear" y crear elementos de narrativa tecnológica.

Paso a paso y Prompts (Generación de Imágenes y Vídeo):

  1. Accedí a la consola de Grok. Previamente elaboré los prompts usando ChatGPT en incógnito para asegurar la terminología técnica fotográfica correcta.
  2. Prompt de Imagen utilizado:
    "A hyper-realistic frontal product shot of a futuristic tech-core streetwear cap, matte black waterproof fabric, tactical buckles, volumetric studio lighting, 8K resolution, solid dark background, clean product photography."
  3. Para los fondos dinámicos de la web, pasamos al modo de vídeo generativo (Image-to-Video).
  4. Prompt de Vídeo animado utilizado:
    "Slow cinematic pan, perfectly static subject, handheld camera movement over a cyberpunk streetwear jacket, heavy rain environment, shallow depth of field, neon reflections on wet fabric, photorealistic 35mm lens."
  5. Descargamos el archivo .mp4 resultante, y lo insertamos en el HTML con:
    <video autoplay loop muted playsinline src="video/asset.mp4"></video>.
⚠ Problema: Deformación Visual en la IA
Obstáculo: Al generar los vídeos de la ropa, la IA tendía a deformar las costuras o derretir los cierres si la instrucción sugería mucho movimiento (efecto alucinación).
Solución: Modifiqué las directrices del prompt introduciendo obligatoriamente las palabras clave "static subject" y aislando el movimiento solo a "handheld camera movement". Al separar las físicas del entorno del objeto, la prenda mantuvo una coherencia geométrica perfecta.

03. Modelado 3D GenAI con Meshy

Meshy.ai fue la llave para dar el salto volumétrico. Convertimos la imagen plana generada por Grok en modelos tridimensionales físicos (mallas topológicas) para usarlos en el entorno de Realidad Aumentada de Unity.

Flujo de Trabajo (Generación y Topología):

  1. Dentro de Meshy, seleccionamos el flujo de trabajo Image to 3D.
  2. Subimos la fotografía plana frontal obtenida en el paso anterior.
  3. Detalle Técnico Clave: Configuramos la salida a High Quality (PBR Textures). PBR (Physically Based Rendering) es vital porque genera mapas de normales (relieve) y rugosidad que harán que la gorra interactúe de forma realista con la luz en Unity.
  4. Exportamos el archivo de forma selectiva en formato .glb (GL Transmission Format Binary). Es el formato más optimizado para empaquetar geometría y texturas en un solo archivo ligero para aplicaciones móviles AR.
⚠ Problema: Modelos Gigantes y Escala Absoluta
Obstáculo: Al importar el `.glb` a Unity, la gorra medía virtualmente decenas de metros, colapsando la cámara espacial y rompiendo el Image Tracking.
Solución: Meshy computa en unidades relativas, mientras que Unity interpreta 1 unidad = 1 metro en la vida real. Apliqué un ajuste en el componente Transform -> Scale del Prefab, reduciéndolo exactamente a 0.05 en los ejes X, Y, Z. Esto redimensionó la prenda a proporciones humanas reales para la simulación móvil.

04. Arquitectura Web y Entorno IDE

🚀 La plataforma central, su diseño de animaciones (SPA) y programación del DOM fueron ensambladas utilizando el motor de Google Antigravity IDE.

Implementación Frontend Avanzada:

  1. Iniciamos el proyecto en Antigravity estructurando el esqueleto definido por los prompts anteriores.
  2. Vinculamos los motores gráficos introduciendo las librerías CDN en el <head>: gsap.min.js y ScrollTrigger.min.js.
  3. Integración de IA en la Web: Para el panel de asistencia interactivo, implementamos Puter.js. Usamos una función asincrónica await puter.ai.chat(historial) atada al DOM, permitiendo al usuario "dialogar" con el entorno sin necesidad de configurar un backend Node.js complejo.
  4. Control del Scroll: Se usó gsap.fromTo() atado a los nodos del DOM para revelar los elementos secuencialmente conforme el usuario hace scroll, inyectando un sentimiento de descubrimiento.
⚠ Problema: Solapamiento Visual y Lag ('Stuttering')
Obstáculo: Al incluir transiciones rápidas y manipular el DOM, el rendimiento caía en dispositivos móviles ("Stuttering") y el puntero personalizado se desfasaba de la posición real.
Solución: Se abandonó la animación de propiedades CSS como margin o width (que fuerzan el Reflow del navegador) en favor de la propiedad Transform. Para el puntero, se programó usando gsap.quickTo(), que interpola valores directamente a la GPU esquivando los cuellos de botella del DOM.
Google Antigravity IDE — script.js
// Resolviendo "Stuttering" y Pizhun Cursor Web Initialization function initInteraction() { const cursor = document.querySelector('.cursor'); // El uso de quickTo evita el reflow del DOM y solventa lag // const xTo = gsap.quickTo(cursor, "x", {duration: 0.2, ease: "power3"}); const yTo = gsap.quickTo(cursor, "y", {duration: 0.2, ease: "power3"}); window.addEventListener('mousemove', e => { xTo(e.clientX); yTo(e.clientY); }); }

05. Realidad Aumentada Extrema en Unity (Tema 13)

Toda la experiencia de captura en el mundo físico transcurre en Unity apoyándose en los motores de AR Foundation para compilar un archivo ejecutable Android (.apk).

Configuración Profunda de la Arquitectura AR:

  1. En el Project Settings, desactivamos la API gráfica Vulkan (que suele generar incompatibilidades con cámaras AR de Android) forzando el uso exclusivo de OpenGLES3.
  2. Desplegamos el AR Session Origin en la escena. Añadimos el script AR Tracked Image Manager.
  3. Creamos una base de datos de imágenes de referencia. Para garantizar que Unity otorgue un rastreo fiable, nos aseguramos de que el marcador físico tuviera alto contraste asimétrico (evitando patrones repetitivos) logrando una puntuación de Tracking superior a 85/100.
  4. Light Estimation (Iluminación Realista): En la cámara de AR, activamos el estimador de luz ambiental. Esto lee los datos del sensor del móvil para ajustar dinámicamente el brillo y la temperatura del material PBR del modelo 3D (la gorra de Meshy), logrando que se integre visualmente en la habitación del usuario.
  5. Requisito Crítico de Interacción: Seleccionamos el Prefab del modelo 3D importado y le añadimos un componente Box Collider. Sin estos colisionadores las mallas 3D son fantasmas matemáticos y la pantalla táctil ignoraría los toques del usuario (Raycasting).
  6. Compilamos estableciendo el Minimum API Level de Android requerido por los paquetes de ARCore.
⚠ Problemas Fatales Frecuentes en el SDK de Unity
Problema 1 - Pantalla Azul y Render Roto: Al limpiar la caché eliminando la carpeta Library del proyecto, el compilador generó un .apk donde la cámara real no se activaba (solo mostraba un fondo azul/negro).
Solución: Todo el 'Universal Render Pipeline' de Unity perdió la sincronía de la cámara. La solución técnica fue ir a los Renderer Features del URP activo y volver a inyectar a la fuerza el script AR Background Render Feature. Esto devolvió el flujo de video real a la pantalla.

Problema 2 - Geometría Desplazada (Tracking de Rostro): El módulo AR sí reconocía el rostro, pero generaba el origen del `.glb` de la gorra dentro de la nariz del usuario, ya que el ancla ARFace se sitúa en el centro geométrico de la cabeza.
Solución: Tuve que alterar el sistema vectorial diseñando un script matemático en C# (ARHatController). Aplicamos un Offset espacial: localPosition = new Vector3(0, 0.15f, 0.05f); subiendo el modelo 15 centímetros hacia el eje Y (frente) y avanzándolo 5 cm en Z, logrando el ajuste anatómico correcto.
PROMPT DE DEPURACIÓN EN CHATGPT:
"En Unity usando AR Foundation, tengo un modelo 3D instanciado. Necesito un script en C# que detecte cuando el usuario toca ese modelo específico en la pantalla táctil de su teléfono y, al hacerlo, abra el navegador en una URL externa. ¿Qué componente físico necesito añadirle al modelo?"

Se activa con el tacto (Raycast implicito) // Exige que el objeto tenga un Box Collider configurado. void OnMouseDown() { // LLama a la API del OS móvil para suspender la app y abrir el navegador web. Scripts Avanzados de Control Espacial

ARHatController.cs - (Solución a la Gorra Flotante del Problema 2). Offset anatómico sobre el Tracking de cara.

using System.Collections.Generic; using UnityEngine; using UnityEngine.XR.ARFoundation; [RequireComponent(typeof(ARFace))] public class ARHatController : MonoBehaviour { private ARFace _arFace; private GameObject _currentHatInstance; void Awake() { _arFace = GetComponent<ARFace>(); } void OnEnable() { _arFace.updated += OnFaceUpdated; } void OnDisable() { _arFace.updated -= OnFaceUpdated; } private void OnFaceUpdated(ARFaceUpdatedEventArgs args) { if (_currentHatInstance != null) { if (_arFace.trackingState == UnityEngine.XR.ARSubsystems.TrackingState.Tracking) _currentHatInstance.SetActive(true); else _currentHatInstance.SetActive(false); } } public void SetHatPrefab(GameObject newHatPrefab) { if (_currentHatInstance != null) Destroy(_currentHatInstance); if (newHatPrefab != null) { _currentHatInstance = Instantiate(newHatPrefab); _currentHatInstance.transform.SetParent(transform, false); _currentHatInstance.transform.localScale = Vector3.one; // EL GRAN ARREGLO AL PROBLEMA 2: // Offset espacial: 15cm hacia arriba y 5cm hacia adelante sobre la frente real. _currentHatInstance.transform.localPosition = new Vector3(0, 0.15f, 0.05f); _currentHatInstance.transform.localRotation = Quaternion.identity; } } }
