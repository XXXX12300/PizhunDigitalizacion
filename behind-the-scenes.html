The Core

Un recorrido detallado por el proceso de diseño y desarrollo de PIZHUN como trabajo práctico de digitalización. Construido explorando la intersección del diseño web y la inteligencia artificial.

01. La Visión Inicial

Este trabajo de digitalización nació con el objetivo de deconstruir la estética tradicional del "streetwear" y crear el diseño conceptual de una supuesta marca ficticia (PIZHUN) con narrativa tecnológica y vanguardista.

En lugar de depender de herramientas tradicionales de vectorización o sesiones de fotos convencionales, el proyecto explora el uso del motor de Grok Imagine para visualizar exactamente qué materiales, luces y texturas conformarían esta marca.

02. Cinematografía Sintética

El dinamismo de la página principal recae fuertemente en su atmósfera audiovisual. Para lograr esos vídeos con acabado de cámara en mano y texturas fotorrealistas de lluvia y asfalto, llevamos los concepts de Grok a herramientas generativas de vídeo.

Las indicaciones técnicas incluían peticiones específicas de lentes (p.ej. "shot on 35mm lens") y movimientos de cámara ("slow pan", "handheld macro shot") para dar la sensación física al robot y a las chaquetas "Tech-Core".

03. Ensamblaje Web

Con las ilustraciones generadas por Grok y los vídeos renderizados, el último paso fue orquestar todo el material en un flujo interactivo.

La entrega del proyecto requería una interfaz que demostrara conocimientos técnicos avanzados, por lo que en lugar de recargar la web con información, implementamos un diseño SPA (Single Page Application) simulado. Se integraron rutinas avanzadas y variables en CSS, además de un control exhaustivo del DOM usando Javascript Vanilla y GSAP.

04. La Experiencia AR en Unity

Para la funcionalidad de Prueba en Realidad Aumentada (la app de Android), pasamos del entorno web a un motor de físicas real.

Importamos los modelos 3D (`.glb`) generados para la web directamente a Unity Engine. Utilizando los paquetes de AR Foundation, combinamos el reconocimiento espacial de la cámara del móvil con la iluminación HD de Unity, permitiendo a la camiseta "flotar" e interactuar con la iluminación de la habitación real en tiempo real antes de exportarlo al `.apk` final descargable.

PASO A PASO

01

Crear las Imágenes con Grok Imagine

Accede a x.ai e inicia sesión con tu cuenta de X (antes Twitter). En el menú lateral selecciona Grok → pestaña Imagen.

  1. Escribe el prompt con el máximo detalle posible. Por ejemplo: "A tech-core streetwear cap, matte black, studio lighting, hyper-realistic, 8K, front view".
  2. Pulsa Generate y espera unos 10-15 segundos.
  3. Si no te gusta el resultado, añade más palabras clave como cinematic, editorial, product photography y vuelve a pulsar Generate.
  4. Cuando tengas la imagen que quieres, descárgala haciendo clic en los tres puntos (⋯) → Descargar.
  5. Repite el proceso para cada producto (cap, chaqueta, camiseta, figura). Guarda todas las imágenes en una carpeta /img dentro de tu proyecto.
02

Generar los Vídeos de Fondo con IA

Los vídeos de la web aportan el movimiento cinematográfico. Para generarlos sigue estos pasos:

  1. Accede a Kling AI o Runway Gen-3 y crea una cuenta gratuita.
  2. Selecciona la opción Image to Video y sube una de las imágenes generadas con Grok.
  3. En el campo de prompt de movimiento escribe: "slow cinematic pan, handheld camera, cyberpunk street, rain, shallow depth of field, 35mm lens".
  4. Selecciona duración: 5 segundos. Resolución: 720p o superior.
  5. Pulsa Generate. El proceso tarda 1-3 minutos.
  6. Descarga el MP4 resultante y guárdalo en una carpeta /video dentro del proyecto.
  7. Repite para cada escena que quieras: el robot, la lluvia en la calle, los detalles de ropa, etc.
03

Construir la Web con HTML, CSS y JavaScript

Toda la web es una SPA (Single Page Application) montada sin frameworks, únicamente con HTML5, CSS3 y JS Vanilla + GSAP para las animaciones.

  1. Crea una carpeta de proyecto y dentro un archivo index.html.
  2. En el <head> importa Google Fonts (Inter + Syne) y enlaza tu style.css.
  3. Añade los scripts de GSAP con el atributo defer para no bloquear el render:
    <script defer src="https://cdnjs.cloudflare.com/ajax/libs/gsap/3.12.2/gsap.min.js"></script>
  4. En style.css, define tus variables de color en :root { --bg-color: #050505; --accent-color: #fff; }.
  5. Crea la estructura de secciones en el HTML: hero, shop, ar-app, brand. Cada una como un <section> con un id único.
  6. Añade las imágenes de productos con loading="lazy" para las que están bajo el fold.
  7. Para los vídeos de fondo usa <video autoplay loop muted playsinline>.
  8. En script.js, implementa el preloader que espera a que se carguen las imágenes principales antes de mostrar la página.
  9. Usa gsap.to() y gsap.fromTo() para las animaciones de entrada del hero.
  10. Implementa el cursor personalizado con gsap.quickTo() para la interpolación fluida del movimiento.
04

App de Realidad Aumentada en Unity

Esta es la parte más técnica del proyecto. Sigue cada paso exactamente en este orden:

  1. Descarga e instala Unity Hub. Desde Unity Hub, instala Unity 2022.3 LTS (versión estable recomendada) con el módulo Android Build Support + Android SDK & NDK Tools.
  2. Crea un nuevo proyecto: New Project → 3D (Core) → Create Project.
  3. Abre el Package Manager (menú Window → Package Manager). En el filtro selecciona "Unity Registry". Busca e instala por este orden:
    • AR Foundation (versión 5.x)
    • ARCore XR Plugin
    • XR Plugin Management
  4. Ve a Edit → Project Settings → XR Plug-in Management. En la pestaña Android, activa la casilla ARCore.
  5. Importa tu modelo 3D: arrastra el archivo .glb directamente a la carpeta Assets en el panel Project. Unity lo convertirá automáticamente a Prefab.
  6. En la escena, elimina la Main Camera por defecto. Añade los componentes AR:
    • Menú GameObject → XR → AR Session
    • Menú GameObject → XR → AR Session Origin
  7. Al objeto AR Session Origin añádele el componente AR Plane Manager (detecta el suelo) y AR Raycast Manager (permite tocar la pantalla para colocar objetos).
  8. Crea un script C# llamado PlaceObject.cs que detecte el toque y coloque el prefab del producto en el punto donde el rayo impacta el plano detectado.
  9. Asigna el prefab del modelo 3D al campo correspondiente del script en el Inspector.
  10. Para el Build: ve a File → Build Settings. Selecciona Android y pulsa Switch Platform. Cuando termine, pulsa Build. Elige una carpeta de destino y espera a que Unity genere el archivo .apk (puede tardar 5-10 minutos).
  11. Instala el APK en tu móvil Android (necesitarás activar "Instalar desde fuentes desconocidas" en Ajustes → Seguridad).
05

Integrar el Asistente de IA con Puter.js

Para dar valor añadido al proyecto integré un chatbot de IA en tiempo real. Aquí el proceso completo de decisión y desarrollo:

  1. Primer intento — OpenRouter: Probé OpenRouter que permite usar modelos como GPT-4 mediante una API. El problema fue que los tiempos de respuesta eran de 5-10 segundos, inaceptables para una tienda. Requería además gestionar API keys y un backend.
  2. Solución — Puter.js: Descubrí Puter.js, que no requiere API keys ni backend. Una sola línea en el <head>:
    <script src="https://js.puter.com/v2/"></script>
  3. Para llamar a la IA, usa puter.ai.chat() pasando un array de mensajes con roles:
  4. Define un system prompt con todo el contexto de tu proyecto como primer mensaje con role: 'system'.
  5. Construye el historial de conversación añadiendo cada mensaje del usuario con role: 'user' y las respuestas con role: 'assistant'.
  6. Crea el panel del chat con CSS (glassmorphism: backdrop-filter: blur(20px)) y un input con un botón de enviar.
  7. Muestra un indicador de carga ("•••" animado) mientras esperas la respuesta y reemplázalo con el texto cuando llegue.
