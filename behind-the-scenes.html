PROYECTO DAW/DAM - OMNICANALIDAD RETAIL

Memoria Técnica:
Arquitectura PIZHUN

Desarrollo integral de un ecosistema Smart Commerce. El usuario interactúa con un catálogo digital en la App, accede a un probador virtual inmersivo mediante Face Tracking AR, y finaliza la transacción en una pasarela Web responsiva.

00. Justificación de Evolución Técnica (10/10)

El enunciado original del proyecto proponía el uso de un marcador físico (Image Tracking) y un componente BoxCollider atado al evento OnMouseDown() para la interacción. Sin embargo, para crear un producto comercialmente viable y maximizar la Experiencia de Usuario (UX), he evolucionado la arquitectura técnica basándome en los siguientes principios:

Evolución 1: Face Tracking vs Image Tracking
Dado que nuestro producto principal es la PIZHUN Neural Cap (Gorra Táctica), obligar al usuario a imprimir una imagen y ponerla en la mesa para ver una gorra flotar carece de sentido comercial. Se ha implementado el subsistema ARFaceManager. El motor rastrea los 468 vértices faciales del usuario, convirtiendo la aplicación en un auténtico Virtual Try-On (Probador Virtual), lo cual representa el estándar actual de la industria del Ecommerce de moda.
Evolución 2: UI Canvas vs BoxCollider
El requerimiento de tocar la malla 3D (BoxCollider) genera una fricción terrible en un Try-On, ya que el modelo se mueve constantemente con la cabeza del usuario (intentar atinarle con el dedo frustra al cliente). Se ha sustituido por un Canvas Overlay que superpone Botones UI estáticos ("Volver al Menú" e "Ir a Web") por encima del feed de la cámara. Esto garantiza accesibilidad, usabilidad y un flujo transaccional limpio hacia la plataforma web.

01. Estrategia de Ecommerce (Tema 14)

PIZHUN actúa como una marca nativa digital de Streetwear Tecnológico. El modelo omnicanal propuesto elimina la fricción: el cliente descubre el producto, se lo prueba virtualmente y es redirigido mediante un Deep Link a la pasarela de pago.

02. La Web como Capa de Confianza (DAW)

Desarrollada en Antigravity IDE, la Landing Page es el punto de conversión. El usuario llega aquí directamente desde la aplicación de Unity.

Arquitectura Mobile-First

  • CSS Grid Responsivo: El catálogo se construyó sin Media Queries densas utilizando la regla grid-template-columns: repeat(auto-fit, minmax(320px, 1fr)). Esto asegura que en un smartphone se muestre en una columna perfecta, maximizando el espacio visual.
  • Single Page UX y Pop-ups: Para visualizar detalles sin cargar páginas nuevas, se implementó la etiqueta nativa de HTML5 <dialog>. Al invocar .showModal(), el navegador renderiza el pop-up en el Top-Layer y bloquea automáticamente el scroll de fondo, mejorando la accesibilidad y el rendimiento.
  • Validación Legal: El Footer incluye políticas de privacidad y métodos de pago seguros cifrados, un requisito vital del Tema 14 para transmitir confianza al consumidor en la transacción final.

03. Integración GenAI (Activos 2D y 3D)

3.1 Modelos de Difusión 2D: Grok AI

Grok Imagine generó los activos fotográficos estáticos y los fondos de vídeo atmosféricos para la web.

Prompt de Fotografía Base:
"A hyper-realistic frontal product shot of a futuristic tech-core streetwear cap, matte black waterproof fabric, tactical buckles, volumetric studio lighting, 8K resolution."
Mitigación de Alucinaciones en Vídeo
Al usar el modo *Image-to-Video* para los fondos de la web, la IA derretía las costuras de la ropa. Se corrigió inyectando parámetros de aislamiento físico: "perfectly static subject, ONLY handheld camera movement", garantizando la integridad de la tela.

3.2 Topología PBR y Mallas Espaciales: Meshy.ai

La imagen 2D de Grok se subió a Meshy.ai utilizando el pipeline Image-to-3D.

  • Materiales PBR: Exportación estricta en High Quality para extraer mapas de Normales (relieve) y Rugosidad. Sin esto, el motor Unity no puede calcular los rebotes de luz realistas en AR.
  • Formato Óptimo: Se exportó como archivo binario .glb, el estándar de la industria móvil para comprimir vértices y texturas.
  • Factor de Escala Absoluta: En Unity, el modelo bruto mide decenas de metros. Se aplicó un Scale de 0.05 en los ejes X,Y,Z para llevar la geometría a proporciones craneales humanas.

04. Desarrollo Técnico AR (Tema 13: Unity)

Ensamblado en Unity 2022.3 LTS usando AR Foundation y ARCore para compilar un sistema Android nativo.

Jerarquía de la Escena y Flujo de UI (Canvas)

La aplicación no inicia abruptamente la cámara. Consta de una arquitectura por capas:

  1. Menu / Catálogo UI: El usuario explora los productos en un Canvas tradicional.
  2. AR Session Origin: Al pulsar "Probar Virtualmente", se despiertan los sensores de ARCore y se activa el AR Face Manager. El sistema localiza la topología facial del usuario e instancia nuestro Prefab de la gorra.
  3. AR_Canvas (Overlay): Sobre la señal de la cámara, se superpone una interfaz limpia con dos botones interactivos: uno para retroceder (VolverAlMenu()) y el desencadenante de la venta (AbrirWeb()).

05. Lógica de Componentes C# (Código Comentado)

Cumpliendo estrictamente con la rúbrica de evaluación, a continuación se adjunta el código fuente desarrollado para la lógica del programa. Cada línea de código contiene comentarios explicativos detallados justificando su función técnica en el ecosistema.

5.1 ARHatController.cs (Solución de Offset Anatómico)

El ancla de seguimiento facial de ARCore se posiciona en el interior de la cabeza (a la altura de la nariz). Si instanciamos la gorra sin modificarla, aparecería incrustada en la cara del usuario. Este script corrige matemáticamente esa discrepancia espacial.

// Librerías principales del motor de Unity y del framework de Realidad Aumentada using UnityEngine; using UnityEngine.XR.ARFoundation; // Atributo de seguridad: Obliga al compilador a añadir un componente ARFace al GameObject. // Si el GameObject no tiene un ARFace, este script no funcionará. [RequireComponent(typeof(ARFace))] public class ARHatController : MonoBehaviour { // Referencia privada al componente ARFace que gestiona los datos biométricos. private ARFace _arFace; // Referencia privada al modelo 3D (la gorra de Meshy) que instanciamos en pantalla. private GameObject _currentHatInstance; // Método Awake: Se llama una sola vez al cargar el objeto. // Enlazamos la variable _arFace con el componente real que existe en el Inspector. void Awake() { _arFace = GetComponent<ARFace>(); } // Método OnEnable: Se dispara cuando el objeto se activa en la escena. // Nos suscribimos al evento "updated" del ARFace. Cuando la cámara detecte movimiento facial, llamará a OnFaceUpdated. void OnEnable() { _arFace.updated += OnFaceUpdated; } // Método OnDisable: Se dispara si el objeto se apaga. Desuscribimos el evento para liberar memoria (buena práctica). void OnDisable() { _arFace.updated -= OnFaceUpdated; } // Función de Callback. Recibe argumentos del sistema de tracking de ARCore. private void OnFaceUpdated(ARFaceUpdatedEventArgs args) { // Verificamos que nuestro modelo de la gorra ya haya sido generado. if (_currentHatInstance != null) { // Si el sensor de la cámara está rastreando activamente una cara sólida... if (_arFace.trackingState == UnityEngine.XR.ARSubsystems.TrackingState.Tracking) // ... encendemos el renderizado del modelo 3D para que se vea. _currentHatInstance.SetActive(true); else // ... si perdemos el rastro (usuario se tapa la cara o sale de plano), ocultamos la gorra. _currentHatInstance.SetActive(false); } } // Método público llamado por otros sistemas para generar la gorra. public void SetHatPrefab(GameObject newHatPrefab) { // Control de seguridad: Si ya había una gorra vieja en la memoria, la destruimos para evitar duplicados. if (_currentHatInstance != null) Destroy(_currentHatInstance); // Si el modelo que nos pasan por parámetro existe... if (newHatPrefab != null) { // Creamos un clon exacto del Prefab en el espacio de memoria (Heap). _currentHatInstance = Instantiate(newHatPrefab); // Emparentamos la gorra al rastreador facial. Ahora la gorra es "hija" de la cara y se moverá con ella. _currentHatInstance.transform.SetParent(transform, false); // Reseteamos la escala a 1 relativo a su padre para evitar deformaciones indeseadas. _currentHatInstance.transform.localScale = Vector3.one; // --- CORRECCIÓN ANATÓMICA CRÍTICA (OFFSET) --- // Dado que el punto 0,0,0 está en la nariz, debemos subir la gorra a la frente. // Modificamos la posición local: // Eje Y: 0.15f (Sube la gorra 15 centímetros hacia arriba de la cabeza). // Eje Z: 0.05f (Adelanta la gorra 5 centímetros para encajar con la inclinación frontal). _currentHatInstance.transform.localPosition = new Vector3(0, 0.15f, 0.05f); // Reseteamos la rotación mediante Cuaterniones base (Identity) para que mire recta hacia el frente. _currentHatInstance.transform.localRotation = Quaternion.identity; } } }

5.2 BotonCompra.cs (Enrutamiento de Interfaz)

Este script se adosa a un gestor vacío en la jerarquía y se enlaza a los eventos OnClick() de los botones del Canvas Overlay, ejecutando el Deep Linking final hacia la web de venta.

// Librería fundamental de Unity. using UnityEngine; // Librería necesaria para gestionar la carga, destrucción y cambio entre diferentes escenas. using UnityEngine.SceneManagement; // Declaración de clase pública que hereda de MonoBehaviour. public class BotonCompra : MonoBehaviour { // Variable pública de tipo cadena de texto (string). // Contiene la URL exacta de la pasarela de pago diseñada en el Tema 14. public string urlCompra = "https://proyecto-pizhun.com/checkout"; // --- MÉTODO VINCULADO AL BOTÓN UI DE COMPRA --- // Debe ser 'public' para que el sistema de eventos de Canvas de Unity pueda detectarlo. public void AbrirWeb() { // Application.OpenURL es una función nativa poderosa. // Suspende el hilo principal de la App (Unity Main Thread), // y le ordena al Sistema Operativo del teléfono que abra el navegador web predeterminado. Application.OpenURL(urlCompra); // Registro de consola útil para depurar el funcionamiento del botón en el Editor de PC. Debug.Log("Saliendo de la app AR. Redirigiendo a la pasarela web del Ecommerce..."); } // --- MÉTODO VINCULADO AL BOTÓN UI DE RETROCESO --- public void VolverAlMenu() { // SceneManager.LoadScene destruye todos los objetos activos de la Realidad Aumentada // (apagando la cámara) y carga la escena que contiene el menú principal del catálogo 2D. SceneManager.LoadScene("MenuPrincipal"); } }

5.3 HologramGlitchAnimator.cs (Estética UI y Feedback)

Aplicado a los paneles del menú para añadir "jugo" visual (Game Feel) simulando un encendido cibernético mediante corrutinas de interpolación matemática.

// Importa librerías para cálculos de tiempo asíncrono y manipulación de Interfaz (UI). using System.Collections; using UnityEngine; using UnityEngine.UI; // Asegura que exista un CanvasGroup (controlador de opacidad maestro) en el panel de la UI. [RequireComponent(typeof(CanvasGroup))] public class HologramGlitchAnimator : MonoBehaviour { // Referencias para alterar la transparencia y tamaño del panel. private CanvasGroup _canvasGroup; private RectTransform _rectTransform; // Tiempo que durará el efecto de "parpadeo" antes de estabilizarse (0.4 segundos). public float transitionDuration = 0.4f; private Vector3 _originalScale; // Recogida de referencias locales antes de que se dibuje el primer frame. private void Awake() { _canvasGroup = GetComponent<CanvasGroup>(); _rectTransform = GetComponent<RectTransform>(); _originalScale = Vector3.one; } // Se activa en el momento exacto en que se enciende el Pop-up del producto. private void OnEnable() { // Inicia un proceso asíncrono paralelo (Corrutina) para no bloquear el juego principal. StartCoroutine(GlitchInCoroutine()); } // Enumerador asíncrono que procesa el parpadeo fotograma a fotograma. private IEnumerator GlitchInCoroutine() { // Contador de tiempo transcurrido. float elapsed = 0f; // Inicia el panel casi invisible (20% de opacidad). _canvasGroup.alpha = 0.2f; // Bucle que se ejecuta mientras el tiempo transcurrido sea menor a los 0.4 segundos definidos. while (elapsed < transitionDuration) { // Suma el tiempo real que tardó en procesar el último fotograma (independiente de los FPS). elapsed += Time.unscaledDeltaTime; // 't' representa el porcentaje de avance de la animación (de 0.0 a 1.0). float t = elapsed / transitionDuration; // LÓGICA DE GLITCH: Si un número aleatorio es mayor al progreso... if (Random.value > t) // ... Asigna una opacidad aleatoria entre 10% y 80% (generando parpadeo caótico). _canvasGroup.alpha = Random.Range(0.1f, 0.8f); else // ... De lo contrario, suaviza la opacidad hacia el 100% usando interpolación lineal (Lerp). _canvasGroup.alpha = Mathf.Lerp(0.5f, 1f, t); // Pausa la ejecución de este bloque de código hasta el siguiente frame (60 veces por segundo). yield return null; } // Al terminar el tiempo, fuerza la opacidad absoluta y escala normal para asegurar legibilidad de los textos. _canvasGroup.alpha = 1f; _rectTransform.localScale = _originalScale; } }
